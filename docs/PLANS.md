# Plans

실행 계획은 제품이나 코드 변경이 한 세션을 넘기거나, 여러 파일/도메인/검증 경로를 건드릴 때 작성합니다.

## 계획이 필요한 경우

- 제품 동작 또는 사용자 여정이 바뀝니다.
- 기술 스택, 아키텍처 경계, 외부 의존성이 바뀝니다.
- 작업이 한 세션 안에 끝나지 않을 가능성이 큽니다.
- 실패 시 되돌리는 경로가 필요합니다.

## 계획 위치

- 활성 계획: `docs/exec-plans/active/`
- 완료 계획: `docs/exec-plans/completed/`
- 기술 부채: `docs/exec-plans/tech-debt-tracker.md`

## 최소 계획 섹션

- 목표
- 범위
- 제외 사항
- 인수 기준
- 검증 방법
- 롤백 또는 되돌림 경로
- 현재 상태
- 다음 행동

## 운영 규칙

- 한 번에 하나의 활성 계획만 기본값으로 둡니다.
- 완료는 체크박스가 아니라 검증 증거로 판단합니다.
- 사용자 프롬프트를 그대로 계획으로 붙여넣지 않습니다. 목표, 범위, 제외 사항, 인수 기준, 검증 방법으로 재구성합니다.
- 계획이 제품 동작을 바꾸면 `docs/product-specs/`를 함께 업데이트합니다.
- 계획이 기술 경계를 바꾸면 `ARCHITECTURE.md` 또는 설계 결정 기록을 함께 업데이트합니다.
