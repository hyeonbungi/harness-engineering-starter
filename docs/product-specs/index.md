# Product Specs

제품 동작의 단일 진실 원천입니다. 이 스타터를 복제한 뒤 실제 프로젝트의 문제, 사용자, 여정, 성공 기준을 이 디렉터리에 기록합니다.

## Documents

- `project-foundation.md`: 첫 제품 정의 템플릿

## Rules

- 사용자 프롬프트를 그대로 저장하지 않습니다.
- 문제, 사용자, 성공 기준, 비목표를 분리합니다.
- 구현 세부 사항은 제품 명세의 결론을 흐리지 않는 범위에서만 둡니다.
- 레거시나 외부 맥락은 실제 프로젝트에서 필요할 때 별도 문서로 분리합니다.
